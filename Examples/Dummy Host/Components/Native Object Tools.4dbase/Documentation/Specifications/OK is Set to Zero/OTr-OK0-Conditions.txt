OTr - OK Variable Behaviour Reference
=======================================
Generated: 2026-04-11
Source: OTr-Phase-009-Spec.md §2.2

This file records, for every public OTr method, whether the legacy OT
counterpart sets the OK system variable, and how OTr replicates that
behaviour. The table is the definitive implementation reference for all
OTr_zSetOK() call sites.

Column key:
  OK=1 success   Y = yes, set explicitly. — = no (OT never did).
  OK=0 error     Y = yes, OTr_zSetOK(0) required on every error path.
                 — = no error path exists.

Where the legacy documentation is silent, the 4D convention is applied:
commands that can fail due to an invalid handle or missing item set OK
to 0 on error; pure query functions that never error (or always return
a sentinel value) do not.

Methods with no legacy OT counterpart (OTr additions) follow the
convention of setting OK to 0 when they have a meaningful failure mode.
