Writes the XML to a temporary file via OTr\_SaveToXMLFileSAX.2. Reads the file back as UTF-8 text.3. Deletes the temporary file. The XML schema produced is identical to that of OTr\_SaveToXML (the DOM equivalent), so the output of both methods is fully interchangeable and can be parsed by OTr\_LoadFromXML / OTr\_LoadFromXMLFile without distinction. NOTE: SAX always produces compact (non-indented) output — 4D's SAX engine provides no built-in indentation facility.  The inPrettyPrint parameter is accepted for API symmetry with OTr\_SaveToXML but has no effect on the text returned. Shadow-type keys (leafKey$type) are included or excluded according to OTr\_IncludeShadowKey (default: True). Use OTr\_SaveToXMLFileSAX to write directly to a named file without the temporary-file round-trip.Attributes: Shared, Invisible, Preemptive indifferent| Parameter | Type |  | Description || --- | --- | --- | --- ||inObject|Integer|➡️|OTr handle||inPrettyPrint|Boolean|➡️|True for indented output; default True (optional)||Result|Text|⬅️|XML string, or empty text if handle is invalid|
