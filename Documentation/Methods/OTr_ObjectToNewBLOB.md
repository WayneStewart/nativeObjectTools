This is the recommended serialisation method because it avoids the by-reference BLOB parameter issues associated with OTr\_ObjectToBLOB.**ORIGINAL DOCUMENTATION***OT ObjectToNewBLOB* serialises an object into a new BLOB which is returned as the function result.Once stored within a BLOB, you must retrieve an object from it with *OTr\_BLOBToObject*, not with *BLOB TO VARIABLE*.If *inObject* is not a valid object handle, an error is generated, OK is set to zero, and an empty BLOB is returned.**Note:** The object remains in memory after serialisation. Clear it with *OTr\_Clear* when no longer needed.Attributes: Shared, Invisible, Preemptive indifferent| Parameter | Type |  | Description || --- | --- | --- | --- ||inObject|Integer|➡️|OTr inObject (handle)||Result|Blob|⬅️|New BLOB containing the serialised object, or an empty BLOB on error|
