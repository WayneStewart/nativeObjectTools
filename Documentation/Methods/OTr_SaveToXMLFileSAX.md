The resulting file is therefore fully interchangeable with files written by OTr\_SaveToXMLFile and can be parsed by OTr\_LoadFromXML / OTr\_LoadFromXMLFile without distinction. NOTE: Unlike the DOM equivalent, SAX always produces compact (non-indented) output — 4D's SAX engine provides no built-in indentation facility.The inPrettyPrint parameter is therefore accepted for API symmetry only and has no effect on the file produced. Shadow-type keys (leafKey$type) are included or excluded according to OTr\_IncludeShadowKey (default: True). Use OTr\_SaveToXMLSAX to obtain the XML as a Text value instead.Attributes: Shared, Invisible, Preemptive indifferent| Parameter | Type |  | Description || --- | --- | --- | --- ||inObject|Integer|➡️|OTr handle||inFilePath|Text   |➡️|Full path of the target file||inPrettyPrint|Boolean|➡️|True for indented output; default True (optional)|
